================================================================
 БЛОК 3 - ПРОФИЛЬНОЕ ПО ПО БИЛЕТУ (зависит от задания)
================================================================

Этот файл - для части экзамена, которая МЕНЯЕТСЯ от билета к
билету: выбор конкретных программ, их установка, базовая
настройка и руководство к одной из них (ГОСТ-шаблон - в файле
03_user_guide_template.odt).


----------------------------------------------------------------
КАК ПОЛЬЗОВАТЬСЯ НА ЭКЗАМЕНЕ
----------------------------------------------------------------

  1. Прочитай раздел "ЗАДАНИЕ" в билете. Выпиши явно ВСЕ
     категории ПО, которые там упомянуты ("необходимо...",
     "потребуется...", "помимо этого...").

  2. Найди фразу "Составить документ «Руководство пользователю»...
     к установленному Вами ..." - это ГЛАВНАЯ программа, на неё
     пишется руководство по ГОСТу. Остальные программы только
     устанавливаются и обосновываются.

  3. Вставь текст задания в промпт ниже и отправь в DeepSeek
     (если на экзамене разрешён интернет/ИИ - уточни правила
     своего колледжа).

  4. Если ИИ недоступен - ищи нужную категорию в списках ниже.


----------------------------------------------------------------
ПРОМПТ ДЛЯ DEEPSEEK (скопируй и доработай)
----------------------------------------------------------------

Ты помогаешь технику-программисту на квалификационном экзамене по
ПМ.04 «Сопровождение и обслуживание программного обеспечения»
(спец. 09.02.07).

ОКРУЖЕНИЕ: виртуальная машина Ubuntu Linux (Desktop), есть sudo и
интернет. Время на установку и настройку профильного ПО - около
25 минут, поэтому варианты должны ставиться быстро (apt/snap/
deb), без сложной компиляции из исходников.

ТЕКСТ ЗАДАНИЯ ИЗ БИЛЕТА (вставь точно как в билете):
"""
[ВСТАВЬ СЮДА ТЕКСТ РАЗДЕЛА "ЗАДАНИЕ"]
"""

Сделай по порядку:

1. Выпиши явный список категорий ПО, которые требуются по тексту
   задания. Не пропускай ни одной фразы про "необходимо/
   потребуется/помимо этого".

2. Для каждой категории предложи ОДНУ конкретную бесплатную/
   open-source программу, которая: работает в Ubuntu Linux;
   ставится через apt/snap/deb без сложной компиляции; является
   распространённым отраслевым решением (это важно для устного
   обоснования выбора).

3. Дай обоснование выбора для каждой программы (2-3 предложения
   в духе: "выбрано, потому что..., совместимо с..., является
   стандартом для...").

4. Дай ТОЧНЫЕ команды установки в Ubuntu для каждой программы,
   по порядку.

5. Дай 2-3 шага базовой настройки после установки (первый запуск,
   язык интерфейса, создание тестового проекта/файла).

6. Явно укажи, какая из этих программ - та самая, упомянутая во
   фразе про "Руководство пользователю" - именно для неё дальше
   нужно писать документ по ГОСТ 19.505-79 (разделы: назначение
   программы; условия выполнения программы; выполнение
   программы; сообщения оператору). Для этой программы
   дополнительно дай: 3-5 её основных функций с шагами
   выполнения и 4-5 типичных сообщений/ошибок программы с
   причиной и действием пользователя - это пойдёт в разделы 3 и
   4 руководства.

7. Предложи 1 идею "настройки обмена данными между установленными
   компонентами" (интеграция двух программ из списка), уместную
   для этого сценария.

8. Если в задании есть требования совместимости ПО с ОС -
   предложи, как продемонстрировать это в Ubuntu (масштабирование
   интерфейса, разрешение экрана, отображение меню/кнопок,
   отключение композиции рабочего стола).

Ответ дай в виде списка по категориям, отдельным блоком - все
команды терминала подряд, без пояснений внутри блока команд
(чтобы можно было скопировать целиком).


----------------------------------------------------------------
ОФЛАЙН-ШПАРГАЛКА ПО КАТЕГОРИЯМ (если ИИ недоступен)
----------------------------------------------------------------

=== Языки и среды выполнения ===

Python 3 + pip
    sudo apt install -y python3 python3-pip python3-venv

Java JDK
    sudo apt install -y openjdk-17-jdk

Node.js
    curl -fsSL https://deb.nodesource.com/setup_lts.x | sudo -E bash -
    sudo apt install -y nodejs

.NET SDK
    sudo apt install -y dotnet-sdk-8.0

Go
    sudo apt install -y golang-go


=== IDE и редакторы (общего назначения) ===

VS Code
    sudo snap install code --classic

Eclipse
    sudo snap install eclipse --classic

IntelliJ IDEA Community
    sudo snap install intellij-idea-community --classic

PyCharm Community
    sudo snap install pycharm-community --classic

NetBeans
    sudo snap install netbeans --classic


=== Embedded / микроконтроллеры (если билет про ARM/прошивки) ===

IDE с JTAG/SWD и профилированием - STM32CubeIDE
    скачать .deb с сайта производителя -> sudo dpkg -i st-stm32cubeide_*.deb

Лёгкая IDE-альтернатива - VS Code + расширение PlatformIO
    sudo snap install code --classic
    (затем установить расширение PlatformIO IDE из Marketplace)

Тулчейн ARM (кросс-компиляция) - GNU Arm Embedded Toolchain
    sudo apt install -y gcc-arm-none-eabi gdb-multiarch

Эмулятор/симулятор МК - QEMU (ARM)
    sudo apt install -y qemu-system-arm

Программатор/отладчик (открытый стек) - OpenOCD
    sudo apt install -y openocd

Контроль целостности ПО / защита доступа к инструментам - AIDE
    sudo apt install -y aide && sudo aideinit


=== DevSecOps / безопасность CI-CD (если билет про SAST/DAST/SCA) ===

SAST (статический анализ) - Semgrep
    python3 -m pip install semgrep

SAST (альтернатива с дашбордом) - SonarQube Community (Docker)
    sudo docker run -d --name sonarqube -p 9000:9000 sonarqube:community

DAST (динамический анализ) - OWASP ZAP
    sudo snap install zaproxy --classic

SCA (сканирование зависимостей) - Trivy
    sudo snap install trivy

Платформа управления уязвимостями - DefectDojo (Docker Compose)
    git clone https://github.com/DefectDojo/django-DefectDojo
    cd django-DefectDojo && ./dc-build.sh && ./dc-up-d.sh

Политики безопасности в Kubernetes - OPA/Gatekeeper или kube-bench
    sudo snap install kubectl --classic

Аудит доступа / шифрование секретов - HashiCorp Vault
    через apt-репозиторий HashiCorp (см. сайт производителя)


=== Базы данных ===

MySQL
    sudo apt install -y mysql-server && sudo mysql_secure_installation

PostgreSQL
    sudo apt install -y postgresql postgresql-contrib

phpMyAdmin
    sudo apt install -y phpmyadmin   (выбрать apache2 при установке)


=== Контейнеризация / версии / web ===

Docker
    sudo apt install -y docker.io && sudo systemctl enable docker

Git
    sudo apt install -y git && git config --global user.name "Имя"

Tomcat
    sudo apt install -y tomcat9 && sudo systemctl enable tomcat9

Apache
    sudo apt install -y apache2

Nginx
    sudo apt install -y nginx


=== Для .deb-файлов, скачанных вручную ===

    sudo dpkg -i имя_файла.deb
    sudo apt-get install -f      (исправить зависимости, если dpkg выдал ошибку)


----------------------------------------------------------------
НАСТРОЙКА ПАРАМЕТРОВ СОВМЕСТИМОСТИ (если нужно по заданию)
----------------------------------------------------------------

Ограниченная цветовая палитра
    Настройки -> Универсальный доступ -> Высокая контрастность
    (вкл/выкл), показать что приложение отображается корректно

Низкое разрешение
    В настройках ВМ временно поставить 800x600, запустить
    приложение, вернуть обратно

Меню и кнопки отображаются корректно
    Запустить приложение, показать интерфейс; при проблемах -
    увеличить масштаб шрифта

Отключение композиции рабочего стола
    gsettings set org.gnome.mutter compositing-manager false

Отключение масштабирования при высоком DPI
    gsettings set org.gnome.desktop.interface scaling-factor 1
    xrandr --dpi 96


----------------------------------------------------------------
ПРИМЕР "НАСТРОЙКИ ОБМЕНА ДАННЫМИ МЕЖДУ КОМПОНЕНТАМИ"
----------------------------------------------------------------

IDE + система контроля версий:
    git init
    git remote add origin <URL>

Файрвол + другое ПО защиты (пример интеграции для embedded/
DevSecOps билетов):
    sudo apt install -y ufw
    sudo ufw allow ssh && sudo ufw allow 80 && sudo ufw enable
    sudo ufw status
