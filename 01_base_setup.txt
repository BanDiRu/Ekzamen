================================================================
 БЛОК 1 + БЛОК 2 — ПОДРОБНЫЙ СПРАВОЧНИК КОМАНД
================================================================

Это справочник для копирования команд по одной, а не скрипт для
запуска целиком. К каждой команде - что она делает и зачем.
Команды разбиты по "окнам" терминала: то, что не зависит друг от
друга, можно выполнять одновременно в разных вкладках/окнах
терминала - так быстрее.


----------------------------------------------------------------
ПОЧЕМУ ИМЕННО ТАК РАЗБИТО ПО ОКНАМ
----------------------------------------------------------------

apt (менеджер пакетов) использует единую блокировку (dpkg lock) -
ДВЕ КОМАНДЫ "apt install" ОДНОВРЕМЕННО РАБОТАТЬ НЕ МОГУТ, вторая
будет ждать или выдаст ошибку "Could not get lock". Поэтому всё,
что ставит пакеты, идёт строго по очереди в одном окне (Окно 1).

Зато создание групп/пользователей, проверка сети, генерация
SSH-ключей - это обычные команды Linux, они НЕ трогают apt и
спокойно работают параллельно, пока в Окне 1 идёт apt update/
upgrade (это самая долгая команда во всём экзамене).

  Окно 1   - Установка всех пакетов через apt (по очереди)
             Когда начинать: сразу, первым

  Окно 2   - Сеть, группы, пользователи, права, SSH-ключи
             Когда начинать: сразу, параллельно с Окном 1

  Окно 3   - Запуск сервисов SSH и CUPS
             Когда начинать: после того как Окно 1 поставит
             openssh-server и cups

  Окно 4   - Бэкап, точки восстановления, мониторинг
             Когда начинать: после того как Окно 1 поставит
             timeshift и logwatch

  ХОСТ     - Снапшоты и экспорт ВМ (НЕ внутри гостевой ОС!)
             Когда начинать: в любой момент - это VirtualBox,
             а не Ubuntu

Открой 2 окна терминала в начале (Окно 1 и Окно 2). Когда Окно 1
закончит ставить пакеты - в нём же (или в новых окнах) выполняешь
Окно 3 и Окно 4 - они тоже не зависят друг от друга и могут идти
рядом.


================================================================
 ОКНО 1 - УСТАНОВКА ПАКЕТОВ (строго по очереди, ничего параллельно)
================================================================

--- 1.1 Обновление системы ---

    sudo apt update

Обновляет локальный список доступных версий пакетов из
репозиториев. Сама ничего не устанавливает и не обновляет -
просто "синхронизирует каталог".

    sudo apt upgrade -y

Обновляет уже установленные пакеты до последних версий. Флаг -y
автоматически отвечает "да" на все вопросы (без него apt будет
переспрашивать). Самая долгая команда - может идти несколько
минут, поэтому запускай её первой и не жди - переходи к Окну 2.


--- 1.2 Guest Additions VirtualBox ---

    sudo apt install -y virtualbox-guest-utils virtualbox-guest-x11 virtualbox-guest-dkms

  * virtualbox-guest-utils - основные службы интеграции с хостом
    (общий буфер обмена, синхронизация времени).
  * virtualbox-guest-x11   - драйвер для авто-подстройки разрешения
    экрана под размер окна ВМ.
  * virtualbox-guest-dkms  - исходники модуля ядра + автоматическая
    пересборка при обновлении ядра.

    sudo reboot

Модуль ядра подгружается только при загрузке системы - без
перезагрузки изменения не применятся. После ребута зайди в
систему и открой терминал(ы) снова.


--- 1.3 SSH-сервер ---

    sudo apt install -y openssh-server

В Ubuntu по умолчанию есть только SSH-клиент (чтобы подключаться
к другим), а сервера (чтобы подключались к тебе) нет - этот
пакет его добавляет.


--- 1.4 Базовое ПО ---

    sudo apt install -y libreoffice p7zip-full p7zip-rar gimp htop hwinfo

  * libreoffice  - офисный пакет (он же понадобится позже, чтобы
    открыть и заполнить руководство пользователя).
  * p7zip-full + p7zip-rar - архиватор с поддержкой 7z/zip и rar.
  * gimp   - растровый графический редактор.
  * htop   - интерактивный монитор процессов и нагрузки (живая
    замена top).
  * hwinfo - вывод подробной информации об оборудовании.


--- 1.5 Виртуальный принтер ---

    sudo apt install -y cups cups-pdf

  * cups     - служба печати (демон, который управляет принтерами
    в системе).
  * cups-pdf - "виртуальный принтер", который вместо бумаги
    создаёт PDF-файл.


--- 1.6 Утилита резервного копирования ---

    sudo apt install -y timeshift

Ставит саму программу. Снапшот этой программой ты создашь позже,
в Окне 4 (после того как пакет уже установлен).


--- 1.7 Модуль политики паролей ---

    sudo apt install -y libpam-pwquality

PAM-модуль, который умеет требовать сложные пароли (минимальная
длина и т.п.) - настраиваешь его в Окне 2 при работе с
аутентификацией.


--- 1.8 Анализатор журналов ---

    sudo apt install -y logwatch

Программа, которая собирает все системные логи за период и
оформляет их в читаемый отчёт - используешь в Окне 4.


  Если хочешь быстрее - все пакеты из 1.2-1.8 можно поставить
  одной командой:

    sudo apt install -y virtualbox-guest-utils virtualbox-guest-x11 virtualbox-guest-dkms openssh-server libreoffice p7zip-full p7zip-rar gimp htop hwinfo cups cups-pdf timeshift libpam-pwquality logwatch


================================================================
 ОКНО 2 - СЕТЬ, ГРУППЫ, ПОЛЬЗОВАТЕЛИ, КЛЮЧИ (параллельно с Окном 1)
================================================================

--- 2.1 Проверка сети ---

    ip addr show

Показывает все сетевые интерфейсы и их IP-адреса. Помогает найти,
какой интерфейс активен (обычно enp0s3) и какой IP выдал NAT.

    hostname -I

То же самое, но короче - просто список IP-адресов в одну строку.

    ping -c 4 google.com

Отправляет 4 пакета до google.com, чтобы подтвердить что DNS и
интернет работают. -c 4 - ограничивает количество пакетов (без
флага ping будет слать пакеты бесконечно, пока не остановишь
Ctrl+C).

    curl -I https://google.com

Запрашивает только заголовки страницы (-I = HEAD-запрос, без
скачивания самой страницы) - подтверждает, что и HTTPS, а не
только ping, работает.


--- 2.2 Группы и права доступа ---

    sudo groupadd developers
    sudo groupadd analysts

Создаёт две новые группы пользователей Linux - чтобы потом
разделить, у кого есть доступ к каким папкам.

    sudo useradd -m -G developers devuser
    sudo passwd devuser

-m - создать домашнюю папку /home/devuser. -G developers - сразу
добавить пользователя в группу developers (дополнительная
группа). passwd запросит ввести пароль два раза.

    sudo useradd -m -G analysts analystuser
    sudo passwd analystuser

То же самое для второго пользователя и второй группы.

    sudo mkdir -p /opt/dev_workspace /opt/analytics_data

Создаёт две папки. -p - не выдавать ошибку, если папка уже есть,
и создавать промежуточные каталоги при необходимости.

    sudo chown :developers /opt/dev_workspace
    sudo chmod 770 /opt/dev_workspace

chown :developers - меняет ТОЛЬКО группу-владельца папки
(двоеточие перед именем означает "меняем группу, а не
пользователя"). chmod 770 - права rwx (чтение/запись/выполнение)
владельцу и группе, и НИЧЕГО посторонним.

    sudo chown :analysts /opt/analytics_data
    sudo chmod 770 /opt/analytics_data

То же самое для второй папки и группы.

    grep -E 'developers|analysts' /etc/group

Проверка: выводит строки файла /etc/group, где встречается слово
developers или analysts - подтверждает, что группы и состав
участников верные.


--- 2.3 SSH-ключи (использовать после того как сервер поднят в Окне 3) ---

    ssh-keygen -t ed25519 -C "exam-key" -f ~/.ssh/id_ed25519 -N ""

Создаёт пару ключей (закрытый+открытый). -t ed25519 - современный
быстрый алгоритм. -C - текстовая метка-комментарий внутри ключа
(для удобства). -f - путь и имя файла ключа. -N "" - пустая
парольная фраза (чтобы не спрашивал её каждый раз при входе -
нормально для учебной ВМ).

    mkdir -p ~/.ssh
    cat ~/.ssh/id_ed25519.pub >> ~/.ssh/authorized_keys
    chmod 700 ~/.ssh
    chmod 600 ~/.ssh/authorized_keys

Добавляет свой публичный ключ в список "разрешённых для входа без
пароля". chmod 700/600 обязательны - SSH откажется работать с
ключами, если права на папку/файл слишком открытые (это его
требование безопасности).

    sudo -u devuser whoami

Выполняет команду whoami от имени devuser, не спрашивая его
пароль (потому что ты root через sudo). Демонстрация
преподавателю, что пользователь реально существует и рабочий.


================================================================
 ОКНО 3 - СЕРВИСЫ SSH И CUPS (после того как Окно 1 поставит пакеты)
================================================================

--- 3.1 SSH ---

    sudo systemctl enable ssh
    sudo systemctl start ssh
    sudo systemctl status ssh --no-pager

enable - служба будет запускаться автоматически при каждой
загрузке системы начиная с этого момента. start - запускает её
прямо сейчас (на случай если enable сам не запустил). status -
проверка, что служба реально active (running); --no-pager - чтобы
вывод не открывался в интерактивном просмотрщике less, который
может "зависнуть".

    ssh devuser@localhost

Подключение к самому себе по SSH - доказывает преподавателю, что
сервер реально принимает соединения (попросит пароль devuser,
либо сработает по ключу, если ты скопировал его в authorized_keys
этого пользователя).


--- 3.2 Виртуальный принтер ---

    sudo systemctl enable cups
    sudo systemctl start cups

Включить автозапуск и запустить службу печати. Дальше проверяешь
в Настройки -> Принтеры - там должен появиться принтер "PDF",
либо просто печатаешь тестовую страницу из любого приложения
(Файл -> Печать -> в PDF).


================================================================
 ОКНО 4 - БЭКАП, ТОЧКИ ВОССТАНОВЛЕНИЯ, МОНИТОРИНГ (после Окна 1)
================================================================

--- 4.1 Резервное копирование (Timeshift) ---

    sudo timeshift --create --comment "Точка восстановления Linux"

Создаёт снимок системы прямо сейчас, с текстовой меткой-
комментарием, чтобы потом было легко узнать, что это за снимок.

    sudo timeshift --list

Выводит список всех созданных снимков - это и есть доказательство
для преподавателя.


--- 4.2 Журнал мониторинга ---

    sudo systemctl status rsyslog --no-pager

rsyslog - служба, которая собирает системные журналы; в Ubuntu
она включена по умолчанию, эта команда просто подтверждает, что
она active (running).

    tail -n 20 /var/log/syslog

Выводит последние 20 строк главного системного журнала -
наглядное доказательство, что логи действительно пишутся.

    sudo logwatch --output stdout --format text --range today

Формирует и сразу печатает в терминал (а не отправляет на
e-mail) читаемый отчёт по всем журналам за сегодняшний день.


================================================================
 ХОСТ - НЕ ВНУТРИ ГОСТЕВОЙ UBUNTU! Это VirtualBox на твоём компьютере
================================================================

Эти команды выполняются в терминале ХОСТА (Windows/Linux/macOS,
где установлен сам VirtualBox), либо просто кликами в окне
VirtualBox - внутри гостевой системы такого инструмента нет.

    VBoxManage export "ИмяВМ" -o exam_appliance.ova

Экспортирует ВМ целиком (диск + настройки) в один файл .ova -
это и есть "установочный образ системы".
Аналог в GUI: Файл -> Экспорт сервиса.

    VBoxManage snapshot "ИмяВМ" take "Точка_1" --description "Состояние после установки ОС"
    VBoxManage snapshot "ИмяВМ" list

Создаёт снимок состояния всей ВМ целиком (не только файлов внутри
Linux, как Timeshift, а вообще всего - диска и памяти). Сделай
минимум 2 таких снимка в разные моменты экзамена.
Аналог в GUI: Машина -> Сделать снимок (Snapshot).


================================================================
 ПОРЯДОК ДЕЙСТВИЙ ЦЕЛИКОМ
================================================================

  1. Открой 2 окна терминала.
  2. В Окне 1 запусти apt update && apt upgrade -y, не дожидаясь -
     сразу переходи в Окно 2.
  3. В Окне 2 делай сеть + группы/пользователей + SSH-ключи, пока
     Окно 1 работает.
  4. Когда Окно 1 освободится - ставь оставшиеся пакеты по очереди
     (1.2-1.8).
  5. После установки и sudo reboot (для Guest Additions) - открой
     Окно 3 и Окно 4 (можно параллельно).
  6. На хосте (отдельно, не в Ubuntu) - сделай экспорт ВМ и
     снапшоты.
